# KetoBuddy — CLAUDE.md

## Prima di fare qualsiasi cosa

Leggi `project-docs/00-index.md` per capire quali documenti esistono e quale leggere in base alla task.

Come minimo, leggi sempre:
- `project-docs/01-project-overview.md` — stack, architettura, decisioni già prese, regole di sviluppo
- Il file milestone attivo (es. `project-docs/MILESTONE_01.md`) — cosa implementare adesso

Leggi anche:
- `project-docs/02-business-logic.md` — se la task riguarda flussi utente, feature flags, monetizzazione
- `assets/config/keto_score_config.json` — solo se la task riguarda lo scoring keto

## Regola principale

Non iniziare a scrivere codice senza aver letto almeno overview e milestone attiva.
